### Network && Protocol
```
ARP (Address Resolution Protocol): IP 주소를 이용해 상대방의 MAC주소를 알아오는 프로토콜이다. 

RARP (Reverse Address Resolution Protocol) : ARP와 반대로 MAC주소를 이용하여 IP주소를 알아오는 프로토콜이다.

GARP (Gratuious Address Resolution Protocol) : 송신 IP와 수신 IP가 동일한 ARP요청으로 자신의 존재를 알리는 목적으로 사용
ex) UP충돌 감지, ARP 테이블 갱신

WSDL Web Service Description Langauage : 웹서비스가 제공하는 서비스에 대한 정보를 기술하기 위한 XML기반 마크업 언어. 

IPv6 : 128bit의 길이를 가짐 16비트씩 8개의 그룹으로 나뉨.
ex) 2001:0db8:85a3:0000:0000:8a2e:0370:7334

IPv4 : 32bit의 길이를 가지고 8비트씩 네부분으로 나뉨
ex) 255.255.255.255

```

# 대칭키 && 공개키
```
대칭키 : 암복호화에 사용하는 키가 동일한 암호화 방식
장점 : 수행 시간이 짧다.
단점 : 안전한 키 교환 방식이 요구되고 사람이 증가할 수록 키 관리가 어려움.

DES Data Encryption Standard : 블록길이가 64비트인 암호(56비트가 키이고 8비트는 검사용) 라운드 횟수는 16회, 한 번의 암호화를 위해 16 라운드를 거친다.

3DES : DES에 대한 전수공격으로 인해 안전하지 않다 판단되어 DES를 보호하기 위해 나온 것
3개읠 키 값을 사용하고, DES 16 라운드의 3배에 해당하는 48라운드를 수행하여 암호화 강도를 높임.

AES Advanced Encryption Standard : 블록 길이가 128비트인 대칭 블록암호, 키 길이는 128/192/256 비트의 가변크기를 제공. 키 크기에 따라 AES-128, AES-192, AES-256 등으로 불림

SEED : 1999년 한국정보진흥원과 국내 암호전문가들이 함께 개발한 알고리즘으로 128비트 비밀키에서 생성된 16개의 64비트 라운드 키를 사용하여 총 16회의 라운드를 거쳐 128비트의 평문 블록을 128비트의 암호문 블록으로 암호화 함.

ARIA Academy Research Institute Agency : 국가보안기술연구소(NSRI) 주도로 학계, 연구소, 정부기관(국정원) 등의 지원을 받아 개발한 국가 암호화 알고리즘
입출력 크기와 사용 가능한 키 크기는 미국 표준 블록암호인 AES와 동일. 

공개키 : 암복호화에 사용하는 키가 서로 다르며 비대칭키 암호화라고도 함. 송수신자 모두 한쌍의 키 (개인키, 공개키)를 갖고 있게 된다.
장점 : 키 분배 필요가 없고, 기밀성, 인증, 부인방지 기능을 제공한다.
단점 : 대칭키 암호화 방식에 비해 속도가 느리다.


```
